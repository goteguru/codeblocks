Lesson 1 – JavaScript Basic Model

Values, Types, Objects, and Functions

Introduction

In this first lesson, we will get acquainted with the fundamentals of JavaScript's way of thinking. The goal is not to understand everything immediately in detail, but to get an idea of what makes up a JavaScript program and how the language "thinks."

Although it may seem mysterious at first, JavaScript is actually not magic: it is a sequence of instructions that work with values. These values are categorized into types, often stored in objects, and many things are handled with functions. We will review these concepts.

What is programming all about?

Programming is essentially formulating instructions for a computer. These instructions are precise, unambiguous, and always executed in the same way.

Typically, a program:

  • receives data (for example, user input),
  • processes it,
  • then returns a result (for example, displays something).

JavaScript (or more broadly, the ECMAScript standard) was originally designed to "bring life" to web pages, but today it is also used on servers, in mobile applications, and many other environments.

What is ECMAScript?

ECMAScript is the official standard name (ECMA-262), which aims to ensure consistency among various implementations (JavaScript, JScript, ActionScript).

ECMAScript languages are very similar; if you know one, you can navigate quite well in any of them. In this course, we will primarily learn the language through JavaScript, since it is the primary programming language of browsers, embedded in all major browsers, so you don't need to install anything—it's available right away!

JavaScript is a dynamic, weakly typed, high-level programming language. Its main characteristics at first glance are:

  • the data type is determined at runtime,
  • the language is flexible,
  • it "overlooks" many small errors for the programmer,
  • yet it allows building complex systems.

Values and Types

In the world of JavaScript, everything is a value. A number, a string, a true/false statement – all are values. JavaScript assigns a type to each value, which indicates what can be done with that value.

Primitive Types

Primitive types are simple, indivisible values. In JavaScript, you will encounter the following primitive types:

number

Any number written as a number is of type number!

In JavaScript, there is no separate integer and floating-point number; everything is a Number. The decimal point is always a dot. This type is accurate to about 15 significant digits (in decimal), so don't expect more precision!

Although it outputs in decimal, it stores numbers in binary (base-2), so what appears as a finite fraction to us might not be one for the computer! For example, 0.1 cannot be represented exactly in binary with finite digits. Since storage is only accurate to about 15 digits, calculations with such values can be imprecise!

The implementation of Number is a floating-point number, specifically a double-precision, 64-bit IEEE-754 floating-point number. (Sounds fancy, right? :)) Besides normal numbers, this format can represent special values like NaN (not a number) or Infinity to indicate that the result is not really a number or is "infinite."

Object and function

In JavaScript, everything that is not primitive is an Object.

What is an Object?

An object is essentially a collection of key-value pairs (called "properties").

In this:

  • name and age are properties,
  • "Anna" and 30 are values.

The value of a property can be any type, including a function (which we call a method, see below) or another object. This allows building hierarchical (multi-level) objects.

Functions themselves are objects, so they can also be values within another object. Thus, objects can be quite complex structures. Most structures in ECMAScript are implemented as such objects. Here in the browser, you access the browser, window, and webpage structure as objects, and in other programs, as data or functions.

Summary

  • JavaScript works with values,
  • values have types,
  • there are primitive and special primitive types,
  • all complex data are Objects,
  • functions play a central role,
  • and if they are part of an object's value, we call them methods.

This is the foundation upon which all further knowledge in the next lessons will be built.
